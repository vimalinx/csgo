# Round 3 功能检查记录（平衡性 + 体验）

## 1) 已修复的功能性 Bug

### Bug A: 联机移动同步读取了不存在的 `game.player.pos`
- 位置: `main.js` 的 `sendPlayerMovement()`
- 问题: `game.player` 未定义，导致联机移动同步被跳过，远端看不到本地实时移动。
- 修复: 改为使用统一玩家位置 `game.pos` 做同步和增量判断。

### Bug B: 聊天热键触发未定义函数 `exitPointerLock()`
- 位置: `setupMultiplayerListeners()` 中 Y/U 聊天快捷键
- 问题: 调用未定义函数会在打开聊天输入时抛异常，影响聊天流程。
- 修复: 改为已存在的 `unlockPointer()`。

### Bug C: 联机计分板金钱读取错误字段
- 位置: `updateMultiplayerScoreboard()`
- 问题: 使用 `game.money`（不存在）导致本地玩家金钱长期显示默认值。
- 修复: 改为 `game.econ.money`。

### Bug D: 返回大厅时清空了全部事件监听
- 位置: `returnToLobby()`
- 问题: `globalEventManager.clear()` 会移除基础输入监听，可能导致回到大厅后再次进局控制异常。
- 修复: 仅清理联机命名空间监听：`globalEventManager.clear('online')`。

## 2) 平衡性检查与建议（先建议，不直接改数值）

### 武器平衡
- 当前伤害与射速结构整体可玩，但建议关注两把“高性价比压制枪”:
  - `scout`（低价高伤）
  - `p90`（大弹匣高容错）
- 建议小步实验（一次只动 1 个参数）:
  1. `scout` 价格上调 `+100 ~ +200`，观察中局出场率。
  2. `p90` 伤害 `22 -> 21` 或价格 `+100`，避免中近距离无脑压制。

### Bot 难度
- 目前难度差主要在 `rpm/spread/damage`，维度单一。
- 建议增加“反应时间”维度（不改基础伤害）：
  1. Easy: 首次见敌后 `220~320ms` 才开火
  2. Normal: `140~220ms`
  3. Hard: `80~140ms`
- 好处: 手感更像人类，难度区分更明显。

### 经济系统
- 当前有基础胜负/击杀奖励，但缺少“连败补偿”会放大滚雪球。
- 建议先做最小版本连败机制（仅 AI 模式验证）:
  - 连败奖励阶梯: `1900/2400/2900/3400`
  - 胜利后重置连败层级

## 3) 用户体验检查与建议

### UI 反馈
- 购买反馈、伤害反馈已有基础实现，信息足够。
- 建议补充:
  1. 聊天打开时状态栏提示 `聊天中（Enter发送 / Esc取消）`
  2. 死亡后明确显示“重生倒计时”（特别是联机）

### 控制流畅性
- 当前射击/换弹/投掷逻辑连贯。
- 建议:
  1. 在设置页增加“鼠标灵敏度”滑杆（目前为固定值）
  2. 购买菜单打开时给当前可购买键位更强高亮（降低新手认知成本）

### 困惑点
- 在线模式购买权限目前不受冻结期限制（设计层面，非本次直接修改）。
- 建议后续由服务器同步 round state 后，再把在线购买限制到 buy phase。
